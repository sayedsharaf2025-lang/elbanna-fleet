# مجموعة بانا لخدمات الأسطول | Pana Fleet Management Group

نظام متكامل واحترافي مصمم خصيصاً لتسيير وإدارة أسطول المركبات والخدمات اللوجستية، الفواتير المالية، العهد والتراخيص، وورش صيانة السيارات التابعة لمجموعة بانا.

An integrated, professional management system designed specifically to streamline and regulate fleet operations, logistics, financial invoicing, vehicle assets, and maintenance workshop tasks for **Pana Fleet Management Group**.

---

## 📂 هيكلة تقسيم الملفات للمشروع | Code Architecture & File Structure

لقد تم تقسيم المشروع بشكل احترافي ومنظم للغاية ليكون مهيئاً للرفع المباشر والعمل على **GitHub** بشكل يطابق أفضل المعايير الهندسية:

```text
├── .gitignore               # الملفات المستبعدة من الرفع إلى GitHub (مثل الحزم ومجلد الأداء)
├── .env.example             # نموذج المتغيرات البيئية لتهيئة الخادم السحابي
├── index.html               # نقطة الدخول والموقع الرئيسي
├── package.json             # ملف الحزم وتحديد المكتبات المعتمدة وأوامر التشغيل
├── tsconfig.json            # إعدادات لغة التايب سكريبت (TypeScript Compiler)
├── vite.config.ts           # إعدادات أداة البناء السريع (Vite) مع مكوّن React
└── src/
    ├── main.tsx             # ملف الدخول الرئيسي لتطبيق React
    ├── App.tsx              # الحاوي الرئيسي (Main App Shell) المعني بحالات الدخول، الملاحة، وحفظ الحسابات الأمنية
    ├── types.ts             # ملف موحد يجمع جميع تعريفات البيانات وأنماط الواجهات (TypeScript Types)
    ├── components/          # مجلد يضم جميع أقسام وموديولات النظام المستقلة:
    │   ├── Dashboard.tsx          # لوحة التحكم التحليلية الشاملة مع الرسوم البيانية والإحصائيات
    │   ├── ViolationsModule.tsx   # موديول تسجيل ومتابعة المخالفات والخصومات الأسبوعية للسائقين
    │   ├── AssetsModule.tsx       # موديول تتبع رخص القيادة وتسيير العهد والتوزيعات الخاصة بالسائقين
    │   ├── ApprovalsModule.tsx    # شاشة إصدار أذونات العمل والرحلات والاعتمادات الرسمية
    │   ├── InvoicesModule.tsx     # موديول الفواتير والتحصيل المالي مع إمكانية استخراج الفواتير وطباعتها
    │   ├── MaintenanceModule.tsx  # إدارة ورش صيانة الأسطول وإصدار الفواتير الفنية المتكاملة
    │   ├── TransportModule.tsx    # موديول تسجيل شحنات النقل البري والخدمات اللوجستية وإسنادها للمركبات
    │   ├── SettingsModule.tsx     # الواجهة المخصصة للتحكم بالتعريف السحابي (Firebase) والاستيراد الجماعي للملفات والبيانات
    │   └── ElBannaLogo.tsx        # شعار الشركة المصمم برمجياً بتقنية SVG
    └── utils/
        └── helpers.ts       # يحتوي على دوال التوليد العشوائي للتعريفات وبيانات التهيئة الأولية (Seed Data)
```

## 🚀 كيفية تشغيل المشروع محلياً | How to Run Locally

### 1. تثبيت الاعتمادات والمكتبات (Install Dependencies)
افتح واجهة الأوامر في مجلد المشروع وقم بتشغيل الأمر التالي لتثبيت كافة المكتبات اللازمة:
```bash
npm install
```

### 2. تشغيل المشروع في وضع التطوير (Run Development Server)
قم بتشغيل خادم التطوير عبر Vite:
```bash
npm run dev
```
سيظهر لك رابط محلي على المتصفح للوصول للتطبيق (افتراضياً على المنفذ `http://localhost:3000`).

### 3. بناء تطبيق الإنتاج (Build and Productize)
لتوليد ملفات الويب الجاهزة للنشر المباشر:
```bash
npm run build
```
سيتم حفظ الملفات الجاهزة في مجلد يسمى `dist`.

## 🛠️ خطوات رفع المشروع لـ GitHub بالتفصيل | Step-by-Step GitHub Upload Guide

اتبع الخطوات التالية بدقة لرفع مشروعك وتأسيس مستودع له على GitHub:

### الخطوة الأولى: تثبيت Git وتهيئة المشروع محلياً
تأكد من تثبيت Git على جهازك، ثم شغّل الأوامر التالية من المجلد الرئيسي للمشروع:
```bash
# 1. تهيئة مستودع Git محلي جديد
git init

# 2. إضافة كافة الملفات للغلاف المؤقت (Staging Area)
# (سيقوم ملف .gitignore تلقائياً باستبعاد مجلد node_modules وملفات الـ dist لتسريع الرفع وحجم المستودع)
git add .

# 3. توثيق وحفظ وحيد البداية للملفات المقسمة برقم تسلسلي
git commit -m "Initial commit: Organized Pana Fleet Management application into split modules"
```

### الخطوة الثانية: إنشاء مستودع جديد على موقع GitHub
1. اذهب إلى حسابك في موقع [GitHub](https://github.com) واضغط على علامة **(+)** ثم اختر **New repository**.
2. أطلق اسماً للمستودع (مثال: `pana-fleet-management`).
3. اجعل المستودع **Public** أو **Private** بحسب تفضيلك.
4. **تنبيه هام:** لا تقم بإضافة أي ملفات افتراضية (No README, No .gitignore, No license) لأننا نمتلك هذه الملفات بالفعل في تطبيقنا محلياً!
5. اضغط على **Create repository**.

### الخطوة الثالثة: ربط مشروعك المحلي بالسحابي ورفع الملفات
انسخ الأوامر التي تظهر لك من موقع GitHub في صفحة المستودع الجديدة لشحن الملفات، ومثالاً عليها كالتالي:
```bash
# 1. تعيين الفرع الرئيسي باسم main
git branch -M main

# 2. ربط مستودعك المحلي بالمستودع البعيد على GitHub (استبدل المعطيات بحساب اسمك ورابط مستودعك)
git remote add origin https://github.com/YOUR_USERNAME/pana-fleet-management.git

# 3. دفع ورفع الملفات بأمان بالكامل إلى GitHub
git push -u origin main
```

مبروك!🎉 سيتم رفع المشروع فوراً بمجلداته المقسمة والمنظمة بشكل نظيف واحترافي تماماً ليكون مرجعاً تقنياً جاهزاً للتطوير أو التقييم المشترك.
